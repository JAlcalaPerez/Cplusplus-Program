#Summarize the project and what problem it was solving.
I created an app that took a users initial investment, monthly deposit, annual interest rate, and number of years and calculated what that would be for balance and interest with and without additional deposits. This was outputted in a table format showing with and without additional deposits.

#What did you do particularly well?
What I felt that I did well was utilizing my h file correctly as well as my use of functions. That really helped the readibility of my .cpp file.

#Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
Looking back on it, I would probably reduce the number of times I use the app.display() function as I think I could have cleaned up the code more. At the time I did feel like it was a good user experience to see what the user had typed in as they were filling out the form but there is likely a more sophisticated and cleaner way to display this compared to what I used. I would also not do calculations in my .main to have it read better and look cleaner.

Working with the .h file was challenging for me as I'm still very much growing in my coding skills so using this many functions and thinking how to structure them took me some time along with the use of constructors. I find zyBooks really helpful in it's examples and hope to still have access to it after the course to come back and revisit some of the concepts. 

#What skills from this project will be particularly transferable to other projects or course work?
